# ООП и объекты

## Задание 1.1

### PyCharm

В PyCharm разработчики могли создать такие классы:

- `Project` - проект пользователя. Поля: название проекта, путь к папке, список файлов, выбранный интерпретатор Python, настройки запуска.
- `File` - файл с кодом. Поля: имя файла, расширение, путь, текст файла, признак сохранения.
- `RunConfiguration` - конфигурация запуска. Поля: имя конфигурации, путь к запускаемому файлу, рабочая папка, аргументы командной строки, выбранный интерпретатор.

### Telegram

В Telegram можно представить такие классы:

- `User` - пользователь. Поля: имя, фамилия, username, номер телефона, статус онлайн.
- `Chat` - чат. Поля: название, список участников, список сообщений, тип чата, закреплённое сообщение.
- `Message` - сообщение. Поля: автор, текст, время отправки, признак прочтения, вложенный файл.

### Компьютерная RPG-игра

В RPG-игре могут быть такие классы:

- `Hero` - герой игрока. Поля: имя, класс персонажа, уровень, здоровье, мана, текущая локация, оружие.
- `Weapon` - оружие. Поля: название, урон, тип оружия, вес, прочность, редкость.
- `Monster` - противник. Поля: имя, вид, здоровье, урон, локация, уровень агрессии, цель атаки.

пример, который показывает побочный эффект от передачи объекта по ссылке:

```python
same_hero = hero
same_hero.level = 4
```

После этого меняется и `hero.level`, потому что переменные `hero` и `same_hero`
ссылаются на один и тот же объект.
